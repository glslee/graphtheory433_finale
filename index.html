Exam Scheduling with Graph Coloring

What is this?

This tool automatically builds an exam schedule so that no two exams conflict, meaning no student is ever forced to take two exams at the same time. It does this using graph coloring, a classical problem in graph theory.

The Graph Model

Formally, a graph is a pair G = (V, E) where V is a set of vertices (nodes) and E is a set of edges, which are pairs of vertices that are connected.

Here, we construct a conflict graph:

  • Each course is a vertex: V = {course₁, course₂, ...}
  • An edge exists between two courses if they share at least one enrolled student: {u, v} ∈ E ⟺ students(u) ∩ students(v) ≠ ∅

Two courses connected by an edge cannot be scheduled in the same time slot.

Graph Coloring & the Chromatic Number

A proper k-coloring of a graph assigns one of k colors to each vertex such that no two adjacent vertices (vertices connected by an edge) share the same color. In our problem, each color is a time slot.

The chromatic number χ(G) is the smallest k for which a proper k-coloring exists. This is exactly the minimum number of exam time slots needed.

Two important bounds on χ(G):

  • Lower bound (clique number ω(G)): A clique is a set of vertices that are all mutually connected. Every vertex in a clique must get a distinct color, so χ(G) ≥ ω(G). In scheduling terms: if 3 courses all share students with each other, you need at least 3 time slots.
  • Upper bound (maximum degree Δ(G)): Brooks' Theorem states that for any connected graph that is not a complete graph or odd cycle, χ(G) ≤ Δ(G), where Δ(G) is the highest degree (number of edges) of any vertex. The greedy algorithm always stays within this bound.

The Greedy Algorithm

Computing the exact chromatic number is NP-hard, meaning no efficient algorithm is known for the general case. Instead, this app uses a greedy heuristic:

  1. Sort all vertices by decreasing degree (courses with the most conflicts go first).
  2. For each vertex in that order, assign the smallest-numbered color not already used by any of its neighbors.

It runs in O(V² + E) time and often achieves or is close to χ(G), but it is not guaranteed to be optimal. The ordering by degree is the key insight: high-degree vertices are the hardest to color, so placing them first gives the algorithm more flexibility.

Special Case: Complete Graphs

If one student is enrolled in every course, every pair of courses conflicts, forming a complete graph Kn (where n is the number of courses). In Kn, every vertex is adjacent to every other, so χ(Kn) = n: you need as many time slots as there are courses, and no compression is possible.
