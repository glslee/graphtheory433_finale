Exam Scheduling with Graph Coloring

What is this?

This tool automatically builds an exam schedule so that no two exams conflict — meaning no student is ever forced to take two exams at the same time.

It does this using a concept from mathematics called graph coloring. Here's the idea:

  1. Model courses as a graph. Each course becomes a node (circle) in the graph. If two courses share even one student, they cannot be scheduled at the same time, so we draw an edge (line) between them. This is called a conflict graph.
  2. Color the graph. Assign a color to each node such that no two connected nodes share the same color. Each color represents a time slot. Because connected nodes have different colors, no two conflicting courses end up in the same slot.
  3. Use as few colors (time slots) as possible. The minimum number of colors needed to color a graph without conflicts is called its chromatic number — and that's the minimum number of exam slots required.

The algorithm: Finding the true chromatic number is computationally hard (NP-hard), so this app uses a fast greedy approximation: courses with the most conflicts are assigned a time slot first, and each course is given the lowest-numbered slot not already used by a conflicting course. This often finds the optimal solution but isn't guaranteed to.
